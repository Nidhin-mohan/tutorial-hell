# Starting point: strong JS, zero DSA, owns an overwhelming course

The learner is a 3-year professional JavaScript developer but is starting **from zero on data structures and algorithms** — they explicitly stated they "don't know anything related to DSA," overriding the milder "rusty" self-assessment. They own the **Namaste DSA** course but find it overwhelming (280+ FANG-level questions, no clear entry point).

Why this matters for future sessions:
- Do **not** assume any prior knowledge of Big-O, recursion, trees, graphs, or standard patterns. Teach each from first principles.
- **Do** assume strong JavaScript fluency — array methods, objects/Map/Set, closures, async. Introduce every DSA idea as JS code first, then generalise.
- **Our job is curation, not content.** Sequence a focused path; point at the relevant slice of the owned course; never dump large problem lists (causes overwhelm). See [[NOTES.md]].
- Target difficulty is LeetCode easy→medium (mid-size/startup), over a 1-2 month ramp. No hard DP / competitive tricks. See [[MISSION.md]].
