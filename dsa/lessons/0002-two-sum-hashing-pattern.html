Two Sum & the Hashing Pattern

The most-asked interview problem of all time — and the first reusable pattern in your toolkit.

⏱ ~15 min 🎯 The win: turn an O(n²) brute force into a clean O(n) 🧩 Pattern: Hashing — "have I seen what I need?"
↩︎ From Lesson 01: Map.get() and Set.has() are O(1). Keep that in your pocket — it's the engine of everything below.

Step 1 · The problem
Two Sum

Given an array of numbers and a target, return the indices of the two numbers that add up to the target. (Exactly one solution exists; you can't use the same element twice.)

twoSum([2, 7, 11, 15], 9)   // → [0, 1]   because 2 + 7 = 9
twoSum([3, 2, 4], 6)         // → [1, 2]   because 2 + 4 = 6

Step 2 · The obvious way (and why it hurts)
Brute force = O(n²)

The instinct: try every pair. For each number, loop through the rest looking for its partner.

function twoSum(nums, target) {
  for (let i = 0; i < nums.length; i++) {        // n times…
    for (let j = i + 1; j < nums.length; j++) {  // …each ~n  → O(n²)
      if (nums[i] + nums[j] === target) return [i, j];
    }
  }
}

It works. But it's the exact loop-inside-a-loop = O(n²) trap from Lesson 01. On a big array, it's slow — and the interviewer will ask you to do better. So how?

Step 3 · The insight
Flip the question around

Instead of asking "what other number pairs with this one?" (which forces a search), ask:

"I need a partner of value target − current. Have I already seen that partner?"
That word — seen — is the trigger. "Have I seen X?" is a membership question, and membership is O(1) with a Map or Set.

So we walk the array once. For each number, we compute the complement (target - num) and ask the Map "have we seen it?" If yes — done. If no — record the current number in the Map and move on. We remember everything we pass, so the partner is always already there by the time we need it.

Step 5 · The solution
The O(n) one-pass answer

function twoSum(nums, target) {
  const seen = new Map();          // value → index

  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];

    if (seen.has(complement)) {      // O(1) — have we seen the partner?
      return [seen.get(complement), i];
    }
    seen.set(nums[i], i);          // remember this number for later
  }
}
Complexity, out loud: "This is O(n) time — one pass, and each Map operation is O(1). And O(n) space, because the Map can grow to hold up to n entries." We traded memory for speed — the most common trade in all of DSA.

Step 6 · The pattern (this is the real lesson)
When to reach for hashing

Two Sum is just one instance. The hashing pattern is the reusable idea. Reach for a Map/Set the moment you catch yourself about to write a nested loop to answer any of these:

  • "Have I seen this before?" → Set (e.g. detect duplicates)
  • "How many times have I seen this?" → Map of value → count (frequency counting)
  • "Where did I see this / its partner?" → Map of value → index (Two Sum)

The tell: any time the brute force is "for each item, search the rest", a hash structure usually collapses that inner search from O(n) to O(1), taking the whole thing from O(n²) to O(n).

  • Trigger phrase: "have I seen…?" / "how many times…?" / "where was…?"
  • Two Sum: one pass, check complement in a Map before inserting current
